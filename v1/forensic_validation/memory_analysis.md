# Memory Analysis Report

## Overview

A memory dump was acquired from the sandbox system while executing the **USB Behavior Simulator** from a removable drive.

The objective was to verify whether **memory forensics can reconstruct the execution chain and detect artifacts generated by the simulator**.

Memory analysis was performed using **Volatility 3**.

---

## Key Finding: USB Executed Script

The command-line analysis revealed execution of a Python script directly from a removable drive.

E:\usb_behavior_simulator.py

This confirms that the payload originated from **drive E:**, representing a **USB device**.

---

## Process Execution Chain

Using `windows.pstree`, the following execution hierarchy was identified:

explorer.exe
└── py.exe
└── python.exe
└── usb_behavior_simulator.py

This indicates the following flow:

1. User interaction through **Explorer**
2. Execution through **Python launcher (`py.exe`)**
3. **Python interpreter** started
4. Script executed from the **USB drive**

---

## Command Line Evidence

Volatility `windows.cmdline` output confirmed the execution path:

py.exe "E:\usb_behavior_simulator.py"
python.exe "E:\usb_behavior_simulator.py"

This provides **direct forensic evidence of the script's origin and execution**.

---

## Observed Memory Artifacts

The simulator intentionally generated several artifacts to validate forensic visibility.

Artifacts verified through **Volatility plugins**:

| Artifact Type | Verification Method |
|---------------|--------------------|
| Process execution | `windows.pslist`, `windows.pstree` |
| Command-line arguments | `windows.cmdline` |
| Loaded modules / DLLs | `windows.dlllist` |
| Open handles | `windows.handles` |

These artifacts demonstrate that **behavioral activity can be reconstructed from memory**.

---

## Forensic Significance

The analysis confirms that memory artifacts can reveal:

- Execution source (**USB drive**)
- Parent-child process relationships
- Command-line execution details
- Runtime process behavior

These findings support the **feasibility of building an automated forensic pipeline** to detect and reconstruct USB-triggered activity.
